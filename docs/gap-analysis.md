# aiburj 功能差距分析 & 改进方案

## 分析来源
对比硅基流动 (siliconflow.cn) 的 10 个页面截图 vs aiburj 当前代码。

---

## 一、当前 aiburj 已有功能

### 用户端 (前端页面)
| 页面 | 路由 | 状态 |
|------|------|------|
| 首页 | `/` | ✅ 基础版 |
| 登录/注册 | `/auth/login`, `/auth/register` | ✅ |
| 控制台 | `/dashboard` | ✅ 接真实 API |
| 模型市场 | `/models` | ✅ 列表 + 搜索 |
| 模型详情 | `/models/[id]` | ✅ |
| API Key 管理 | `/api-keys` | ✅ |
| 用量统计 | `/usage` | ✅ |
| 充值 | `/recharge` | ✅ |

### 管理后台
| 页面 | 路由 | 状态 |
|------|------|------|
| 管理首页 | `/admin` | ✅ 基础统计 |
| 用户管理 | `/admin/users` | ✅ 列表 |
| 模型配置 | `/admin/models` | ✅ CRUD |
| 充值审核 | `/admin/recharge` | ✅ 基础 |

### 后端 API (23 个端点)
- 认证：login, register, me
- 计费：balance, usage, orders, api-keys CRUD
- 模型：list
- 代理：chat/completions, completions, models (OpenAI 兼容)
- 充值：plans, recharge, orders
- 管理：stats, users CRUD, models CRUD

## 二、硅基流动有的、aiburj 缺失的功能

### 2.1 用户端缺失

| 功能 | 重要度 | 说明 |
|------|--------|------|
| **文档中心** | ⭐⭐⭐⭐⭐ | API 参考、SDK 指南、快速开始。硅基流动有完整的文档站 |
| **个人中心/设置** | ⭐⭐⭐⭐ | 修改密码、绑定手机、通知偏好、个人资料 |
| **首页改版（已登录）** | ⭐⭐⭐⭐ | 硅基流动登录后首页：余额卡片 + 快捷入口 + 用量概览 + 热门模型推荐 |
| **推荐官/拉新系统** | ⭐⭐⭐ | 邀请链接 + 返佣 + 推广数据面板 |
| **模型对比** | ⭐⭐⭐ | 并排比较 2-4 个模型的价格、能力、参数 |
| **体验中心/Playground** | ⭐⭐⭐ | 在线测试模型，无需写代码 |
| **通知中心** | ⭐⭐⭐ | 余额不足提醒、模型上下线通知、系统公告 |
| **搜索增强** | ⭐⭐ | 模型搜索 + 文档搜索全局化 |
| **模型收藏** | ⭐⭐ | 收藏常用模型，快速访问 |

### 2.2 管理后台缺失

| 功能 | 重要度 | 说明 |
|------|--------|------|
| **运营数据看板** | ⭐⭐⭐⭐⭐ | 实时图表：调用量趋势、收入曲线、模型热度排名、用户增长 |
| **系统日志** | ⭐⭐⭐⭐ | API 调用日志、错误日志、管理员操作审计 |
| **公告管理** | ⭐⭐⭐⭐ | 发布系统公告、模型上下线通知 |
| **套餐管理** | ⭐⭐⭐⭐ | 动态配置充值套餐（当前是硬编码） |
| **模型监控** | ⭐⭐⭐ | 上游可用性检测、延迟监控、自动熔断 |
| **用户详情页** | ⭐⭐⭐ | 点击用户查看完整画像：调用历史、余额变动、API Key 列表 |
| **批量操作** | ⭐⭐ | 批量启用/禁用用户、批量调价 |
| **系统设置** | ⭐⭐ | 全局参数：默认限流、新用户赠送金额、支持邮箱 |

## 三、改进方案（分阶段）

### Phase 6 — 增强（本次要做的）
**定位：完善当前中转站体验，补齐核心缺失**

1. **首页改版**（已登录）
   - 余额卡片（醒目显示）
   - 快捷操作入口（创建 Key / 查看文档 / 体验模型）
   - 本月用量概览
   - 最近使用的模型

2. **文档中心** `/docs`
   - API 快速开始
   - 模型列表及参数
   - SDK 示例（Python / JS / curl）
   - 计费说明

3. **个人中心** `/profile`
   - 修改密码
   - 个人资料
   - 通知设置

4. **管理后台增强**
   - `/admin` → 运营数据看板（图表）
   - `/admin/users` → 增强：用户详情弹窗、搜索筛选
   - `/admin/models` → 增强：批量操作、状态监控
   - `/admin/recharge` → 增强：套餐管理
   - **新增** `/admin/logs` → 系统日志
   - **新增** `/admin/settings` → 系统设置

### Phase 7 — 聚合（后续迭代）
- Playground 在线测试
- 模型对比
- 推荐官拉新
- 通知系统

## 四、架构决策

> **当前定位：API 中转站**
> 用户用 aiburj 的 Key → aiburj 转发到上游厂商 → 返回结果
> 
> **未来定位：API 聚合平台**
> 统一计费、统一限流、智能路由、多厂商容灾

本次 Phase 6 仍然保持中转站架构，但为将来聚合做准备：
- 用量统计预留 `provider` 字段
- 管理后台预留模型健康监控接口
- 文档结构预留 SDK 多语言支持
